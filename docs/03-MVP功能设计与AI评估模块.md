# 0-12月龄婴幼儿成长管理小程序 — MVP功能设计与AI评估模块

## 一、用户画像

### 画像A：新手宝妈小雅
- 年龄：26岁，一线城市，第一胎
- 宝宝月龄：4个月
- 特征：焦虑感强，每天刷育儿知识，担心宝宝发育不达标
- 核心诉求：知道宝宝是否正常、该做什么、怎么做
- 使用场景：喂奶间隙、宝宝睡后、儿保体检前后

### 画像B：二胎宝妈阿琳
- 年龄：32岁，二线城市，第二胎
- 宝宝月龄：7个月
- 特征：有经验但时间紧，需要高效工具
- 核心诉求：快速记录、对比老大老二数据、不遗漏关键节点
- 使用场景：碎片时间记录、出门在外时查看

## 二、MVP功能列表（V1.0）— 5大核心模块

基于Kano模型与四象限分析，MVP选择以下5个模块：

### 模块1：成长数据记录

**功能描述：** 支持快速记录宝宝身高、体重、头围三大体格指标。输入数据后自动生成WHO生长曲线对比图，直观显示宝宝在同龄群体中的百分位水平。支持历史数据查看与趋势分析。

**用户流程：**
```
首页 → 点击"记录" → 选择指标类型 → 输入数值 → 自动计算百分位 → 生成曲线 → 保存
```

**数据埋点：**
| 指标 | 说明 |
|------|------|
| record_daily_rate | 日活用户中完成记录的比例 |
| record_type_distribution | 各指标记录频次分布 |
| record_avg_duration | 单次记录平均耗时（秒） |
| curve_view_rate | 记录后查看生长曲线的比例 |

**使用场景：** 小雅带宝宝做完4月龄儿保，医生说体重偏轻。她打开小程序，输入体重数据，看到宝宝从25百分位掉到了15百分位，曲线图清晰显示了下降趋势。她截图保存，下次儿保时给医生看。

---

### 模块2：发育里程碑对照

**功能描述：** 基于AAP（美国儿科学会）和中国卫健委《0-6岁儿童发育行为评估量表》，按月龄展示大运动、精细动作、语言、认知、社交五大领域的发育里程碑。宝妈可通过勾选宝宝已掌握的技能，生成发育雷达图，识别优势领域和待关注领域。

**用户流程：**
```
首页 → "发育评估" → 选择宝宝月龄 → 系统展示该月龄应掌握技能列表
→ 勾选已掌握项 → 生成五维雷达图 → 查看评估结果与建议
```

**数据埋点：**
| 指标 | 说明 |
|------|------|
| milestone_check_rate | 月活用户中完成里程碑勾选的比例 |
| milestone_completion_avg | 平均勾选技能数 / 应掌握技能数 |
| radar_view_rate | 查看雷达图的比例 |
| domain_weak_detection | 各维度标记为"待关注"的频率 |

**使用场景：** 阿琳的宝宝7个月了，她打开里程碑对照，发现宝宝大运动（独坐、翻身）都达标了，但精细动作（捏取小物件）还没掌握。系统用黄色标记"待关注"，并推荐了3个锻炼精细动作的小游戏。

---

### 模块3：睡眠/喂养/便便记录与分析

**功能描述：** 提供极简的"一键打卡"式记录。睡眠支持开始/结束计时，喂养支持母乳（左/右/时长）/奶粉（ml）/辅食（种类+量），便便支持性状、颜色、频次。每日/每周自动生成统计报告，识别异常模式（如突然夜醒增多、奶量下降）。

**用户流程：**
```
首页 → 点击快捷记录按钮 → 选择类型（睡眠/喂养/便便）
→ 快速录入 → 自动保存 → 查看今日统计 / 本周趋势
```

**数据埋点：**
| 指标 | 说明 |
|------|------|
| log_frequency | 人均日记录条数 |
| sleep_log_rate | 睡眠记录占比 |
| feed_log_rate | 喂养记录占比 |
| report_view_rate | 查看统计报告的比例 |
| anomaly_alert_click | 异常提醒点击率 |

**使用场景：** 小雅发现宝宝最近夜醒频繁，她打开睡眠记录查看，发现过去一周夜间醒来从2次增加到5次。系统提示"宝宝夜间觉醒频次较上周增加150%，可能与出牙/猛涨期有关"，并附上应对建议。

### 模块4：AI智能评估与天赋分析（核心差异化功能）

**功能描述：** 基于用户记录的多维数据（体格发育、里程碑达成、睡眠模式、喂养数据、互动记录），利用AI模型生成个性化的宝宝"天赋倾向评估报告"和"发展方向建议"。这是产品的核心差异化功能。

#### 4.1 AI评估模型架构

```
输入数据层
├── 体格数据：身高/体重/头围曲线走势
├── 发育数据：五大领域里程碑达成情况
├── 行为数据：睡眠节律、喂养偏好、情绪表现
├── 互动数据：亲子游戏参与记录、反应速度
└── 环境数据：月龄、喂养方式、家庭结构

    ↓ 特征提取与编码

分析引擎层
├── 发育水平评估模型（对标WHO/AAP标准）
├── 天赋倾向识别模型（五维天赋画像）
├── 发展轨迹预测模型（趋势外推）
├── 风险预警模型（异常检测）
└── 培养建议生成模型（基于知识图谱+LLM）

    ↓

输出层
├── 发育水平报告（各维度评分 + 同龄对比）
├── 天赋倾向画像（雷达图 + 文字解读）
├── 发展方向建议（3条可执行建议）
├── 每日互动推荐（个性化游戏/活动）
└── 风险预警提示（如需要，建议就医）
```

#### 4.2 五维天赋画像

| 天赋维度 | 评估依据 | 典型表现 |
|----------|----------|----------|
| 运动天赋 | 大运动里程碑达成速度、活动量、身体协调性 | 抬头早、翻身快、喜欢蹬腿、精力旺盛 |
| 语言天赋 | 发音频次、对声音的反应、咿呀学语的丰富度 | 爱发声、对语音敏感、模仿声音快 |
| 逻辑认知 | 对因果关系的理解、探索行为、注意力持续时间 | 好奇心强、爱观察、重复实验行为 |
| 艺术感知 | 对色彩/音乐/节奏的反应、情绪表达丰富度 | 听音乐会安静、对色彩鲜艳物品感兴趣 |
| 社交情感 | 眼神交流、微笑频率、对陌生人反应、共情表现 | 爱笑、认人早、社交主动性高 |

#### 4.4 AI技术方案

| 组件 | 技术选型 | 说明 |
|------|----------|------|
| 数据存储 | 微信云开发 + 云数据库 | 用户数据加密存储，符合隐私法规 |
| 特征工程 | Python + scikit-learn | 标准化、编码、特征选择 |
| 发育评估模型 | 规则引擎 + 逻辑回归 | 基于WHO/AAP标准的规则匹配 |
| 天赋倾向模型 | 随机森林 / XGBoost | 多维特征 → 五维评分 |
| 报告生成 | LLM（GPT-4o / 通义千问） | 自然语言报告生成，温度控制在0.3 |
| 推荐引擎 | 知识图谱 + 协同过滤 | 月龄×天赋→匹配游戏/活动库 |
| 数据埋点 | 微信自带 + 自定义事件 | 全链路用户行为追踪 |

**数据埋点（AI模块）：**
| 指标 | 说明 |
|------|------|
| ai_report_generate_count | AI报告生成次数 |
| ai_report_view_duration | 报告平均查看时长 |
| ai_report_share_rate | 报告分享率 |
| ai_suggestion_click_rate | 建议项点击率 |
| ai_game_recommend_play | 推荐游戏的实际参与率 |
| ai_data_input_count | 触发AI分析所需的最少数据条目 |

**使用场景：** 小雅记录了宝宝3个月的数据后，系统提示"数据已足够生成首次AI评估报告"。她点击查看，发现宝宝在运动和社交维度得分较高，系统建议多做音乐互动来补充艺术感知。她把报告分享到宝妈群，引来其他妈妈的好奇和下载。

### 模块5：疫苗接种提醒与管理

**功能描述：** 根据宝宝出生日期和月龄，自动匹配国家免疫规划疫苗接种时间表，提前3天推送提醒。支持记录已接种疫苗、标记接种反应、生成接种档案。

**用户流程：**
```
首次使用 → 输入宝宝出生日期 → 自动匹配疫苗时间表
→ 到期前3天推送提醒 → 接种后标记"已完成"
→ 记录接种反应（可选）→ 生成接种档案
```

**数据埋点：**
| 指标 | 说明 |
|------|------|
| vaccine_remind_click | 提醒推送点击率 |
| vaccine_mark_rate | 标记已接种的比例 |
| vaccine_reaction_log | 接种反应记录率 |
| vaccine_profile_view | 查看接种档案的比例 |

## 五、隐私合规设计

| 合规要求 | 实现方式 |
|----------|----------|
| 不强制收集位置信息 | 定位功能默认关闭，仅用于查找附近儿保机构时可选开启 |
| 数据加密存储 | 云数据库字段级加密，传输走HTTPS |
| 用户数据可删除 | 设置页提供"一键删除所有数据"功能 |
| 未成年人数据保护 | 遵循《个人信息保护法》关于14岁以下未成年人的规定 |
| 第三方数据共享 | 默认不共享，分享报告时需用户确认 |
| AI模型训练 | 默认不使用用户数据训练模型，需用户主动授权 |
