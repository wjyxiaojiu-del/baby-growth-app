# 0-12月龄婴幼儿成长管理小程序 — 用户调研问卷设计

## 一、调研目标

- 明确0-12月龄宝宝妈的核心痛点与高频需求
- 量化各功能需求的优先级权重
- 识别"新手宝妈"与"二胎宝妈"的需求差异
- 为MVP功能选择提供数据支撑

## 二、抽样设计

| 维度 | 要求 |
|------|------|
| 样本量 | 300份有效问卷 |
| 边际误差 | ±2% |
| 置信水平 | 90% |
| 分层要求 | 新手宝妈 ≥ 50%，二胎宝妈 ≥ 30%，其余20%不限 |
| 宝宝月龄覆盖 | 0-3月、4-6月、7-9月、10-12月各 ≥ 20% |
| 回收周期 | 7天 |

## 三、投放渠道与激励

| 渠道 | 目标份数 | 投放方式 | 激励机制 |
|------|----------|----------|----------|
| 母婴社群（微信群/妈妈帮） | 150份 | 社群KOL转发 | 完成问卷抽微信红包（1-5元） |
| 育物公众号/小红书 | 100份 | 图文嵌入问卷链接 | 关注公众号领取育儿电子手册 |
| 线下儿保科/社区卫生中心 | 50份 | 二维码海报 | 现场扫码赠送婴儿湿巾小样 |

## 五、问卷质量控制机制

| 控制项 | 策略 |
|--------|------|
| 防重复提交 | 同一微信ID限填1次 |
| 最短答题时间 | < 60秒标记为无效问卷 |
| 关键字段校验 | Q1/Q2/Q9/Q14为必填，空白则无法提交 |
| 反向题目 | Q8中设置反向选项（"功能太少"）用于检测一致性 |
| 逻辑跳转 | Q6选"从不记录"则跳至Q9，减少无效作答 |
| IP/设备检测 | 同一设备提交超过2次则屏蔽 |

## 六、数据编码与分析框架

### 6.1 定量分析

- Q14功能评分 → 计算每项功能的加权平均分 → 排序
- Q9痛点排序 → 给第1名3分、第2名2分、第3名1分 → 累加得分
- 交叉分析：按Q1（月龄段）×Q14（功能评分）查看需求差异

### 6.2 定性分析（开放题）

- 对Q19/Q20进行关键词提取与聚类
- 建立"痛点-功能"映射矩阵
- 每个聚类赋予权重值（出现频次 × 情感强度）

### 6.3 Kano模型分类

| 需求类型 | 判断标准 | 产品策略 |
|----------|----------|----------|
| 基本型（Must-be） | 不具备时用户极度不满，具备时认为理所当然 | MVP必须包含 |
| 期望型（One-dimensional） | 具备时满意，不具备时不满 | MVP优先包含 |
| 兴奋型（Attractive） | 具备时惊喜，不具备时无所谓 | V2.0/V3.0迭代 |
| 无差异型 | 无论有没有都无所谓 | 不开发 |
