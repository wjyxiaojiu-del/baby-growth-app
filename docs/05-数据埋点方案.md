# 0-12月龄婴幼儿成长管理小程序 — 数据埋点方案

## 一、埋点设计原则

- **最小必要原则**：只采集产品迭代必需的数据，不采集位置、通讯录等敏感信息
- **用户知情原则**：首次使用时弹出隐私协议，明确告知数据用途
- **可删除原则**：用户可在"我的→隐私设置"中一键删除所有行为数据

## 二、全链路埋点清单

### 2.1 用户生命周期事件

| 事件名 | 触发时机 | 参数 | 用途 |
|--------|----------|------|------|
| user_register | 首次登录 | source(渠道), timestamp | 渠道归因分析 |
| user_profile_complete | 完善宝宝信息 | baby_month_age, feed_type,胎次 | 用户画像构建 |
| user_active | 每日首次打开 | date, session_count | DAU计算 |
| user_retention_7d | 注册后第7天是否活跃 | is_active | 7日留存率 |
| user_churn | 连续14天未打开 | last_active_date | 流失预警 |

### 2.2 核心功能使用事件

| 事件名 | 触发时机 | 参数 | 分析价值 |
|--------|----------|------|----------|
| record_create | 创建一条记录 | record_type(height/weight/head/sleep/feed/poop), duration_sec | 功能使用频率 |
| record_edit | 编辑已有记录 | record_type, edit_count | 数据修正行为 |
| record_delete | 删除记录 | record_type | 记录意愿/纠错 |
| milestone_check | 完成里程碑勾选 | month_age, checked_count, total_count, completion_rate | 发育关注程度 |
| milestone_detail_view | 查看某项技能详情 | skill_id, domain(motor/language/cognitive/social/art) | 关注维度分析 |
| report_generate | 触发AI报告生成 | data_days_count, record_count | 数据充足度 |
| report_view | 查看AI报告 | view_duration_sec, scroll_depth_pct | 报告吸引力 |
| report_share | 分享报告 | share_channel(friend/moments/save_image) | 传播力分析 |
| suggestion_click | 点击AI建议项 | suggestion_type(game/nurture/checkup) | 建议有效性 |
| vaccine_remind_click | 点击疫苗提醒 | remind_advance_days | 提醒有效性 |
| vaccine_complete | 标记已接种 | vaccine_name, is_on_time | 接种依从性 |

### 2.3 用户体验事件

| 事件名 | 触发时机 | 参数 | 分析价值 |
|--------|----------|------|----------|
| page_view | 页面浏览 | page_name, from_page, duration_sec | 页面流量/路径 |
| onboarding_complete | 完成新手引导 | step_count, skip_count | 引导完成率 |
| error_occur | 接口/功能报错 | error_code, page_name, action | 稳定性监控 |
| feedback_submit | 提交反馈 | feedback_type, content_length | 用户声音收集 |

## 三、核心看板指标

### 3.1 健康度指标

| 指标 | 计算方式 | V1.0目标 | 基准线 |
|------|----------|----------|--------|
| DAU | 每日user_active去重 | 100+ | - |
| 7日留存 | D7活跃/D0注册 | 30% | 行业均值25% |
| 人均记录条数 | 日记录总数/DAU | 1.5 | - |
| 功能渗透率 | 使用某功能的用户/DAU | 记录60%, 评估30% | - |
| 报告生成率 | 生成报告用户/注册用户 | 20% | - |
| 分享率 | 分享用户/查看报告用户 | 10% | - |

### 3.2 AI模块专项指标

| 指标 | 说明 | 优化方向 |
|------|------|----------|
| report_trigger_rate | 数据充足后触发报告生成的比例 | 优化触发引导 |
| report_avg_view_duration | 报告平均查看时长 | 提升内容质量 |
| report_scroll_depth | 报告页面滚动深度 | 优化信息层级 |
| suggestion_action_rate | 建议项实际执行率 | 提升建议可操作性 |
| game_recommend_play_rate | 推荐游戏的参与率 | 优化推荐匹配度 |
| report_regenerate_rate | 重新生成报告的比例 | 评估报告满意度 |

## 四、数据看板搭建建议

使用微信云开发自带的数据分析 + 第三方工具（如GrowingIO或自建Metabase）：

```
看板层级：
├── 概览看板：DAU/WAU/MAU、新增、留存曲线
├── 功能看板：各模块使用频率、漏斗转化
├── AI看板：报告生成量、查看时长、分享率
├── 质量看板：错误率、接口响应时间
└── 用户看板：月龄分布、喂养方式分布、城市分布
```
